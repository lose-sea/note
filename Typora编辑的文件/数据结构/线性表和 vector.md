# 线性表

线性表是 n 个具有相同特征的数据元素的有序序列

表头，表尾，前驱，后继， 空表

## 线性表的顺序存储– 顺序表

---

线性表的顺序存储就是顺序表

顺序表可以用数组来实现，而数组有两中不同的申请方式

因此，顺序表的实现方式也有两种

1. 数组采用动态申请的方式—-动态顺序表（new ， delete）
2. 数组采用静态申请的方式 —- 静态顺序表

### 动态实现

通过 new 和 delete 来申请和释放空间

```c++
int *a; // 接受 new 出来的数组的地址
int capacity;   // 标记当前数组的大小 
int n;			// 标记有效元素的个数
```

优点: 

1. 按需索取,需要多少空间就申请多少

缺点: 

1. 动态实现会进行空间的申请与释放以及数据的拷贝
2. 而这些操作是非常耗时的

例如:向顺序表末尾依次插入1到10

第一次空间不够,扩容

```c++
	a = new int[4] 
	capacity = 4;  
```

第二次空间不够,扩容

```c++
	// 第二次空间不够,扩容
	int* t = new int[capacity * 2]; 
	memcpy(t, a, sizeof(int) * capacity); 
	delete[] a; 
	a = t; 
	capacity *= 2; 
```

第三次空间不够,扩容

```c++
	// 第三次空间不够,扩容
	int* t = new int[capacity * 2]; 
	memcpy(t, a, sizeof(int) * capacity); 
	delete[] a; 
	a = t; 
	capacity *= 2;
```

通过 new 和 delete 来申请和释放内存

### 静态实现

---

直接创建一个足够大的数组,来充当顺序表

```c++
int a[10000]; 
int n; 
```

缺点:

1. 一旦空间占满,信赖的数据就会溢出
2. 如果为了保险而申请很大的空间,数据量小的情况下,会浪费很多空间
