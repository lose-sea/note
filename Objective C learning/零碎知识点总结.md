# 面向对象和面向过程

**面向过程: ** 以函数为基本单位 ,把问题拆解成一系列步骤,按照顺序依次进行, 程序 = 数据结构 + 算法

**面向对象: **: 以对象为基本单位, 将数据和行为封装在一起, 通过对象之间的交互来解决问题, 程序 = 对象 + 消息 . 特点: 封装, 继承, 多态



# 对象和指针

我们通过一个指针对对象进行访问, 我们所有的指针对象化都是通过对象的指针变量来访问它. 

**这里的对象保存在堆内存中** 

# self关键字

让类的方法访问类的另一个方法或成员变量

1. **实例方法（- 开头）**：`self` 指向**调用该方法的具体实例对象**，谁调用方法，`self` 就代表谁，类型固定为当前类 / 子类的实例，具体对象由调用者决定。
2. **类方法（+ 开头）**：`self` 可正常使用，指向**调用该方法的类本身（Class 对象）**，而非实例对象，常用于类层面的逻辑（如创建实例、访问类属性）。



当局部变量与成员变量重名时, 局部变量会隐藏成员变量, **但是可以通过self 来强制调用成员变量**

 # id 

id类型的变量, 是动态绑定的, 任意类的对象都可以赋值给id 类型的变量

**id这个类型是在运行的时候判别该对象所属的类, 而且是在运行时动态绑定, 不是在编译的时候就确定要调用的方法**

# 形参可变的方法

在定义方法时, 在最后一个形参后面增加`, ...`, 这就代表它可以接受好多参数

**在最后一个参数后面一定要传入一个nil, 保证指针为空可以退出**

> **个数可变的形参只能处于形参列表的最后, 也就是说, 一个方法中最多只能有一个长度可变的形参** 
>
> + `va_list argList; ` 声明一个可变参数列表变量
>
> + `void va_start(va_list ap, last);`  
>
>   + > + **`ap`**：`va_list`类型的变量，用于存储可变参数列表的指针。
>     > + **`last`**：函数中**最后一个固定参数**（可变参数列表前的那个参数），`va_start` 会以这个参数的地址为基准，定位到第一个可变参数的位置。
>
> + `type va_arg(va_list ap, type); ` 
>
>   + > + ap 必须是**已经通过 `va_start` 初始化**的变量（和 `va_end` 用的是同一个）；每次调用 `va_arg`，`ap` 会自动 “移动” 到下一个参数的位置 
>     >
>     > + 必须是**具体的、完整的数据类型**，不能是别名 / 不完整类型；且必须和实际传入的参数类型一致，否则会导致内存错误
>
> + `void va_end(va_list ap);` 
>
>   + > ap 就是通过va_List创建的参数列表变量, 且必须通过va_start 初始化 

**个数可变的形参只可以处于形参列表的最后, 也就是说, 一个方法中最多只有一个长度可变的形参**

# 访问控制符

+ @private : 只允许在这个类内部访问, 用来彻底隐藏成员变量
+ @protect : 只允许类和子类访问
+ @package: **这个访问控制符, 可以在当前类和类的同一镜像中访问, (就是编译后的同一框架或同一个执行文件), 这一般就是考虑到,在开发一个框架中, 其他类或者其他函数也要用到这个成员变量, 我们就考虑使用这个访问控制符, 这些类, 函数就都在一个映像中(包括主函数). **
+ @public : 可以在任何地方进行访问

**访问控制符只能用来修饰成员变量**

# 特性指示符 

+ assign: 指定对属性只是进行一个简单的赋值, 不更改对所赋的值的引用计数 
+ atomic (nonatomic): 指定合成的存取方法是否为一个原子操作. 
+ copy: 
+ string/ weak: 表示对于被赋值对象持有强引用还是弱引用, 他们决定对象是否被自动回收

# 继承

子类继承父类的全部成员变量 . 全部方法(包括初始化方法) 

**无论父类接口部分的成员变量使用何种访问控制符修饰, 子类的接口定义的成员变量都不允许与父类接口部分的成员变量重名(如果重名会被覆盖, 但可以通过调用父类的getter方法访问)(super只能调用父类的方法, 不能调用父类的成员变量)**

# 类的多态

oc的指针类型有两个, 一个是编译时候的类型, 一个是运行时的类型, 编译的类型有声明该变量时使用的那种类型来决定,如果编译的时候和运行的时候类型不一样,就有可能出现多态 

```objc
int main(int argc, const char * argv[]) {
    @autoreleasepool {
        Base* a = [[Base alloc] init];
        Sub* b = [[Sub alloc] init];
        [a base];
        [a test];
        [b test];
        [b sub];
        Base* c = [[Sub alloc] init];
        [c base];
        [c test];
//        [c sub];   // 报错
        id d = c;
        [d sub];
        [d base];
        [(Base*)d test];
    }
    return EXIT_SUCCESS;
}
```

输出:  

```objc
父类的base
父类的say
子类的test
子类的sub
子类的test
父类的base
子类的sub
父类的base
子类的test
```

> `**c` 无法调用 `sub` 的原因: `c` 的声明类型是 `Base*`，编译器在编译阶段只看声明类型：**

```objc
Base* c = [[Sub alloc] init];
[c test];  // ✅ Base 有 test 声明，编译通过，运行时调 Sub 的 test
[c sub];   // ❌ Base 没有 sub 声明，编译直接报错，到不了运行时
  
id d = c;
[d sub];   // ✅ id 跳过编译期检查，运行时找到 Sub 的 sub，成功
```

当我们讲一个子类的对象赋给父类的指针变量 ,他的行为方法总是表现出子类方法的特征, 而不是父类的行为特征, 他也只能调用父类的方法, 但是父类的方法呈现的却是子类的行为.

# 强制类型转换

除了id类型的变量之外, 指针变量只能调用它编译的时候的方法, 如果需要让这个指针变量调用它运行的时候的类型方法, 这必须要强制类型转换成运行时候的类型. 

**强制类型转换只是改变了编译时期的类型, 但没有改变它实际所指向的类型, 如果调用编译时期的类型的话就会报错** 



```objc
        Sub* e = [[Base alloc] init];
        [e base];
        [e test];
//        [e sub];    // 报错
//        [(Sub*) e sub]; // 报错
```

可以把子类的对象赋给父类的指针变量, 但是不能把父类的类型赋给子类的对象.  因为子类拥有父类的一切, 父类没有子类的方法

向一个 `Base` 对象发送了 `sub` 消息，但 `Base` 根本没有 `sub` 方法！ 

> e 的实际指向是 Base 类型
>
> + 编译器看 `e` 是 `Sub*`，认为可以调 `sub`，**编译通过**
> + 但运行时实际对象是 `Base`，`Base` 没有 `sub`，**崩溃**



把子类对象直接赋给父类的指针变量, 实际指向的是子类, 再赋给 id 类型, 实际上指向仍是子类, 调用 base 是继承于父类的, 但是如果子类重写了父类的一个方法, 再去调用, 就会报错, 因为 id 不知道该参考父类的还是子类的

# 协议



在OC中为了增加灵活性，新增了两个关键字。

**@optional**:实现类可以选择是否实现这些方法, 如果没有该关键字, 只要声明了协议, 不实现就会报错

**@require**:位于该关键字之后、@optional或@end之前声明的方法是必需的，实现类必需实现这些方法。

默认是require
