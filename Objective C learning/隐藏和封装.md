

封装是面向对象的三大特征之一, (另外两个是继承 和 多态), 它指的是将对象的状态信息隐藏在对象内部 , 不允许外部程序直接访问对象内部信息, 而是通过该类所提供的方法来实现对内部信息的操作和访问 

## 访问控制符 

Objective-C提供了4个访问控制符, @private, @package, @protected, @public, 分别代表了4个访问控制级别. 

+ @private (当前类访问权限), 如果类的成员变量使用@private访问控制符来限制, 则这个成员只能在当前类的内部被访问, **这个访问控制符用于彻底隐藏成员变量, 在类的实现部分定义成员变量相当于默认室友给你这种访问权限** 

+ @package (与映像访问权限相同): 如果类的成员变量使用@package访问控制符来限制, 则这个成员可以在当前类以及当前类实现的同一个映像的任意地方访问. 很显然,这个访问控制符用于部分用于隐藏成员变量

+ @protected (子类访问权限): 如果类的成员变量使用@package 访问控制来限制, 则这个成员可以在当前类. 当前类的子类的任意地方访问. 很显然, 这个访问控制符用于部分暴露成员变量.在类的接口部分定义的成员变量默认使用这中访问权限.

+ @public (公共访问权限): 这是一个最宽松的访问控制级别, 如果类的成员变量使用@public 限制, 这个成员变量可以在任意地方访问,不管是否处于同一映像中, 也不管最后是否具有父子继承关系 

  ```objective-c
  #import<Foundation/Foundation.h>
  @interface FKPerson : NSObject {
  @private
      NSString* name;
      int age;
  }
  // 提供方法来操作name age
  - (void) setName: (NSString*) name;
  - (NSString*) name;
  - (void) setAge: (int) age;
  - (int) age;
  @end
  
  @implementation FKPerson
  - (void) setName: (NSString*) name {
      if ([name length] > 6 || [name length] < 2) {
          NSLog(@"你设置的人名不符合要求");
      } else {
          self->name = name;
      }
  }
  - (void) setAge: (int) age {
      if (age > 100 || age < 0) {
          NSLog(@"您设置的年龄不合法");
      } else {
          self->age = age;
      }
  }
  - (NSString*) name {
      return name;
  }
  - (int) age {
      return age;
  }
  @end
  
  
  int main(int argc, const char* argv[]) {
      @autoreleasepool {
          FKPerson* p = [[FKPerson alloc] init];
          [p setAge: 1000];
          NSLog(@"%d", [p age]);
          [p setAge: 30];
          NSLog(@"%d", [p age]);
          [p  setName: @"鑫研"];
          NSLog(@"%@", [p name]);
      }
      return 0;
  }
  ```

  输出: 

  ```objective-c
  您设置的年龄不合法
  0
  30
  鑫研
  ```

  在上面的FKPerson类中, 该类的name 和 age两个成员变量只能在FKPerson类内采可以操作和访问

  Objective-C类中成员变量的setter 和 getter 方法有非常重要的意义,(setter方法命名为成员变量名的首字母, 并在前面分别增加set动词, getter 方法名为成员名去掉下划线前缀), **如果一个Objective-C类的每一个成员变量都被使用@private限制, 并为每个成员变量都提供了 settert 和 getter方法, 那么这个类就是一个符合规范的类**

## 合成存取方法

Objective-C自动合成了setter 方法个 getter方法, 这样开发人员就可以避免书写乏味的seter. getter方法, 而且如果用户需要控制某个getter 或 setter方法的实现, 开发者依然可以提供setter或getter方法, 这个用户提供的setter 或getter方法将覆盖系统自动合成的setter或getter方法

 **让系统自动合成setter 和 getter方法只要如下两步** 

1. 在类接口部分使用@property指令定义属性, 使用@property定义时无须放在类接口部分的花括号里,而是直接放在@interface. @end之间定义, @property指示符放在属性定义的最签名
2. 在类实现部分使用@synthesize指令声明该属性即可 

在采用上述两个步骤合成存取方法之后, 不仅会合成成对的setter 和 getter方法, 还会自动在类实现部分定义一个与getter方法同名的成员变量, 如果为某个类定义了一个成员变量, 并提供了相应的setter getter方法, 那么可称定义了一个属性 (property) 

Xcode编码规范推荐将成员变量名定义为以下划线开头, 通过Xcode生成的模板代码中可以看到如下代码

```objective-c
@synthesize window = _window; 
```

上面代码用于告诉系统合成的property对应的成员变量为_window, 而不是window;  

```objective-c
@synthesize property名 [= 成员变量名]; 
```



```objective-c
#import<Foundation/Foundation.h> 

@interface FKUser : NSObject 
    // 使用@property定义了 3 个property 
@property (nonatomic)NSString* name;
@property NSString* pass;
@property NSDate* birth; 
@end; 

@implementation FKUser
    // 为3个property合成setter和getter 方法
    // 指定name property 底层对应的成员变量名为 _name
@synthesize name = _name;
@synthesize pass;
@synthesize birth;
    
// 实现自定义的setName: 方法, 添加自己的控制逻辑
- (void) setName: (NSString*) name {
    self->_name = [NSString stringWithFormat: @"+++%@", name];
}
@end
```



> @synthesize name = _name;
> @synthesize pass;
> @synthesize birth; 
>
> - 编译器默认规则：会自动为每个 `@property` 合成 `_属性名` 格式的成员变量（比如 `name` → `_name`、`pass` → `_pass`、`birth` → `_birth`）。
> - 如果不需要给成员变量**自定义命名**（重命名），这些 `@synthesize` 语句**完全可以不写**，编译器会自动完成所有必要的合成工作。

当使用@property定义property时, 还可以在 @property 和类型之间用括号添加一些额外的指示符, 可使用的特殊指示符如下: 

> **assign:** 该指示符指定对属性只是进行简单赋值, 不更改对所赋的引用计数, 这个指示符只要适用于NSInteger等计数类型, 以及short, float.double. 结构体等各种C]数据类型 

(计数引用时Objective-C内存回收的概念, 当一个对象的引用计数大于 0 时, 表明该对象还不应该被回收, 由于NSInteger等基础类型, 以及short. Float, double, 结构体等各种C数据类型都不存在回收问题, 因此使用assign即可)  

> **atomic (nonatomic):** 指定合成的存取方法是否为原子操作, 所谓原子操作, 主要指是否线程安全, 如果使用 atomic , 那么合成的存, 取方法都是线程安全的 – 当一个线程进入存, 取方法的方法体后, 其他线程无法进入该存, 取方法,这样就可以避免多线程并发破坏对象的数据完整性, atomic是默认值, 虽然 atomic可以保证对象数据的完整性, 但 atomic 的线程安全会造成性能下降, 因此, 大多数单线程环境下, 我们都会考虑使用nanetomic来提高存取方法的访问性能 
>
> **copy: **如果使用copy指示符, 当调用setter方法对成员变量赋值时, 会将被赋值的对象复制一个副本, 再将该副本赋值给成员变量. copy指示符会将原来成员变量所引用对象的引用计数减一, 当成员变量的类型是可变类型, 或其子类时可变类型时, 被赋值的对象有可能在赋值之后被修改, 如果程序不需要这种修改影响 setter 方法设置的成员变量的值, 此时就可以考虑使用copy 指示符 

```objective-c
@interface FKBook : NSObject
@property (nonatomic) NSString* name;
@end

@implementation FKBook
@synthesize name;
@end

int main(int argc, const char* []) {
    @autoreleasepool {
        FKBook* book = [[FKBook alloc] init];
        NSMutableString* str = [NSMutableString stringWithString: @"疯狂iOS讲义"];
        [book setName: str];
        NSLog(@"book 的name为: %@", [book name]);
        [str appendString: @"is a good book"];  // 修改str字符串
        NSLog(@"book 的name为: %@", [book name]);
    }
    return 0;
}
```

输出: 

```objective-c
book 的name为: 疯狂iOS讲义
book 的name为: 疯狂iOS讲义is a good book
```



name的类型是NSString类型, 而NSString有一个NSMutable 子类, 当我们把一个NSMutable对象赋值给Book的name属性之后, NSMutable对象可能被修改 – 由于定义name是未使用copy指示符, 因此, NSMutable对象的修改将会影响FKBook的name属性 

如果对上面的程序进行修改, 将定义name属性的一行增加copy指示符, 

`@property (nonatomic copy) NSString name; `

这样当程序执行[book setName]; 时, 程序会将str指向的NSMutable 对象复制一个副本, 再将副本作为setName: 的参数值, 这样就可以实现当程序通过str修改底层NSMutable对象时, FKbook的name属性值不会随着改变

> getter, setter: 这两个指示符用于为合成的getter方法, setter方法指定自定义的方法名, 例如: getter = abc指定getter方法名为abc , setter = xyz: , (由于setter方法要带参数,所以不要忘记冒号), 则指定 setter 方法的方法名为xyz: . 

```objective-c
@interface FKItem : NSObject
@property (assign, nonatomic, getter = wawa, setter = nana: )int price;
@end

@implementation FKItem
@end

int main(int argc, const char* argv[]) {
    @autoreleasepool {
        FKItem* item = [[FKItem alloc] init];
        [item nana: 30];
        NSLog(@"item 的 price = %d", [item wawa]);
    }
    return 0;
}

```

> readonly, readwrite: readonly指示系统只合成gettet方法 ,不在合成setter方法 ,readwrite时默认值, 指示系统需要合成setter, getter方法
>
> retain: 使用retain 指示符定义属性时, 当把某个对象赋值给该属性时, 该属性原来所引用的对象的引用计数减一, 被赋值对象的引用计数加一 

```objective-c
@interface FKwin : NSObject
@property (nonatomic, retain) NSDate* date;
@end

@implementation FKwin

@end

int main(int  argc, const char* argv[]) {
    FKwin* win = [[FKwin alloc] init];
    NSDate* date = [NSDate date];
    // 第一次赋值, data的引用计数为1
    NSLog(@"date的引用计数为: @ld", date.retainCount);
    [win setDate: date];
    // 下面输出的引用计数为2
    NSLog(@"date的引用计数为%ld", date.retainCount);
    // 释放 date 的引用计数, date的引用计数减一
    [date release];
    // 下面输出的引用计数为1
    NSLog(@"[win date] 的引用计数为: %ld", (unsigned long)[win date].retainCount);
    
    return 0;
}
```

> 1. **第一次输出（计数为 1）**
>
>    当通过`[NSDate date]`创建`date`对象时，系统会为该对象分配内存并初始化，此时对象的引用计数默认是**1**（表示有 1 个指针指向它）。
>
> 2. **第二次输出（计数为 2）**
>
>    把`date`赋值给`FKWin`对象的`data`属性时，因为该属性用`retain`修饰，会触发`retain`方法：
>
>    - `retain`的作用是**增加对象的引用计数**，确保对象不会被意外释放。
>    - 因此`date`的引用计数从 1 变为**2**（`date`本身的指针 + `FKWin`的`data`属性指针，共 2 个指针指向它）。
>
> 3. **第三次输出（计数为 1）**
>
>    调用`[date release]`时，`release`方法会**减少对象的引用计数**：
>
>    - 此时`date`的引用计数从 2 减为**1**。
>    - 但`FKWin`的`data`属性仍持有该对象，所以对象不会被销毁。

- **`retain`的意义**：为对象 “续命”，只要引用计数 > 0，对象就不会被系统回收。
- **`release`的意义**：放弃对对象的所有权，引用计数减到 0 时，对象会被销毁并回收内存。
- **属性修饰符`retain`**：赋值时自动对新值调用`retain`，对旧值调用`release`，是 MRC 中管理对象属性的核心方式。

> strong, weak: strong指示符指定该属性对赋值对象持有强引用,而weak指示符指定该属性对被赋值对象持有弱引用. 强引用的意思是, 只要该强引用指向被赋值的对象, 那么该对象就不会自动回收; 弱引用的意思是, 即使该弱引用指向被赋值的对象,该对象也可能被会回收
>
> unsafe_unretained: 这个指示符与weak指示符基本相似, 对于只被unsafe_unretained指针所指向的对象, 该对象也可能被回收, 与weak指针不同的是, 当unsafe_unretained指针所引用的对象被回收后, unsafe_unretained指针不会被赋值为nil, 因此可能导致程序崩溃, 一般来说, 使用unsafe_unretained指示符不如使用weak 

## 使用点语法访问属性

点语法是一种非常简单的用法, 点语法只是一种简化写法, 其本质依然是调用getter 和 setter 方法. 当程序调用点语法获取指定对象的属性值时, 本质上就是获取对象的getter 方法的返回值.因此, 只要该对象有getter 方法 (无论该对象是否存在对应的成员变量), 程序就可以通过点语法来获取属性值; 当程序调用点语法设置对象的属性值时, 本质上就是调用对象的setter方法进行设置.因此, 只要该对象有setter方法 (无论该对象是否存在对应的成员变量), 程序就可以通过点语法来设置对象的属性值. 

```objective-c
@interface FKCard : NSObject
@property (nonatomic, copy) NSString* flower;
@property (nonatomic, copy) NSString* value;
@end

@implementation FKCard

@end

int main(int argc, const char* argv[]) {
    @autoreleasepool {
        FKCard* card = [[FKCard alloc] init];
        card.flower = @"A";
        card.value = @"A";
        NSLog(@"我的扑克牌为 %@, %@", card.flower, card.value);
    }
    return 0;
}
```
